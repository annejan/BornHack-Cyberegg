# BornPets mini-games

> **See also:** [README.md](README.md) for project overview, [GAME.md](GAME.md) for player-facing instructions, [CLOCK.md](CLOCK.md) for watch/alarm/calendar docs, [CONTACTS_SCREEN.md](CONTACTS_SCREEN.md) for the meshcore chat UI.

The BornPets game screen ships with seven mini-games.  Each one is
opened from the Play modal (`Game screen → Play → <name>`), runs as
a full-screen takeover, and is exited with **Cancel**.

## Why play

Winning a mini-game — or, for the open-ended ones, finishing a
session — calls `lifecycle::award_inspiration(MiniGame::<game>)`,
which:

- Reduces the pet's `drained` stat (a one-shot relief — the longer
  match-3 games scale this by score).
- Sets a per-game cooldown so winning one game doesn't lock the
  others.

The game becomes available again in the Play modal once its
cooldown ticks back to zero.  Items that are on cooldown render
greyed out in the modal with the time remaining shown next to the
label.

## Common controls

| Button             | Action across games                                              |
| ------------------ | ---------------------------------------------------------------- |
| **Cancel**         | Always closes the active mini-game and returns to the game grid. |
| **Up/Down/Left/Right** | Move a cursor (when there is one) or scroll choices.         |
| **Fire / Execute** | Confirm / place / activate.  Game-specific where they differ.    |

A few games use **Execute** as a separate "secondary" action — those
deviations are called out per game below.

## Tic-tac-toe

Classic 3×3 grid against a built-in AI.

### Mechanic
You play **X**, the AI plays **O**.  Three in a row (horizontal,
vertical, or diagonal) wins.  Difficulty is picked at the start of
each game — the picker is the first thing you see when you open
Tic-tac-toe.

### Difficulty
- **Normal**: the AI plays optimal minimax most of the time, with a
  35 % chance per move of slipping in a random legal move instead.
  Beatable, but rewards thinking ahead.
- **Impossible**: pure minimax, no slips.  At best you draw.

### Controls
| Button         | Effect                                              |
| -------------- | --------------------------------------------------- |
| Up/Down/Left/Right | Move the cursor across cells.                  |
| Fire / Execute | Place an X at the cursor / confirm difficulty pick. |
| Cancel         | Exit.                                               |

### Winning
Player wins or draws → inspiration awarded.  AI wins → no reward.

## Lights Out

5×5 grid of lights.  Toggling a cell flips that cell **and** its
four orthogonal neighbours.  The puzzle is solved when every light
is off.

### Mechanic
The starting layout is shuffled randomly but always solvable
(generated by toggling random cells from the all-off state, so the
inverse sequence will solve it).  No move limit — keep at it
until you've turned everything off.

### Controls
| Button         | Effect                       |
| -------------- | ---------------------------- |
| Up/Down/Left/Right | Move the cursor.         |
| Fire / Execute | Toggle the cursor cell.      |
| Cancel         | Exit (no reward if unsolved).|

### Winning
All lights off → inspiration awarded, game closes.

## Nim

Misère Nim with the classic 4 rows of `[1, 3, 5, 7]` sticks.

### Mechanic
You and the **EI** alternate turns.  On each turn, take **one or
more sticks from a single row**.  Whoever is forced to take the
**last stick on the board loses**.

Each move is two-phase:

1. **Row select** — Up/Down moves a dithered cursor between rows.
   Press Fire to commit the row.
2. **Count select** — Right shrinks the take-count, Left grows it
   (the rightmost sticks turn dithered grey to mark them for
   removal).  Press Fire to take them.

### Difficulty
- **Normal**: 35 % chance per EI move of playing a random legal
  move instead of the optimal Nim strategy.
- **Hard**: always optimal — XOR-strategy for standard Nim,
  switching to the "leave an odd count of 1-piles" rule once
  every pile has ≤1 stick (misère endgame).

### Controls
| Button         | Effect                                              |
| -------------- | --------------------------------------------------- |
| Up/Down        | Row select / change count (depending on phase).     |
| Left/Right     | Change take-count (in count-select phase).          |
| Fire / Execute | Commit the current row / count.                     |
| Cancel         | Exit.                                               |

### Winning
Player avoids taking the last stick → inspiration awarded.

## Maze

An 18×18 procedurally-generated maze rendered fullscreen.  Find an
exit on any side.

### Mechanic
The maze is generated by Recursive Backtracker (randomised DFS), so
every cell is reachable from every other.  Multiple exits may exist
along the border.

Cells you've visited are marked, so it's clear at a glance where
you've been.

### Controls
| Button         | Effect                                                      |
| -------------- | ----------------------------------------------------------- |
| Up/Down/Left/Right | Move one cell in that direction (if the wall allows).   |
| Fire / Execute | Trigger the cheat / regen action (see the source for the secret). |
| Cancel         | Exit.                                                       |

### Winning
Walk into an exit on the maze border → inspiration awarded.

## Black Hole

A six-row triangular pyramid (21 circles).  Position-evaluation
puzzle adapted to a tiny board.

### Mechanic
You (white tokens with black numbers) and the AI (black tokens
with white numbers) alternate placing the numbers **1..10** in
your own colour, **in order** — both place "1" first, then both
"2", and so on.  After 20 placements, one cell remains empty: the
**black hole**.

Each player's score is the **sum of their numbers in cells
adjacent to the hole**.  Lowest score wins.

### Controls
| Button         | Effect                                  |
| -------------- | --------------------------------------- |
| Up/Down/Left/Right | Move the placement cursor.          |
| Fire / Execute | Place your current number at the cursor.|
| Cancel         | Exit.                                   |

### Winning
Lower adjacent-sum than the AI → inspiration awarded.

## Triple Born

BornHack-themed reskin of the *Triple Town* match-3-and-merge
mechanic on a 6×6 grid.

### Mechanic
A **preview tile** is shown.  Place it on the grid; whenever three
matching tiles end up in a 4-connected group, they **merge into the
next tier**, which can immediately re-merge upwards.  Wildcards
become whatever neighbour they're touching and may cascade.

Cars (the analogue of the original game's bears) move randomly each
turn and become **wrecks** if they get trapped — wrecks have their
own merge series.

### Tile lineup
(1) Grass → (2) Bush → (3) Tree → (4) Small tent → (5) Big tent →
(6) Village → (7) Festival camp → (8) BornHack → (9) CyberÆgg.

Vehicle series (when cars get trapped):
Wreck → Junkyard → Scrapheap.

### Controls
| Button         | Effect                                               |
| -------------- | ---------------------------------------------------- |
| Up/Down/Left/Right | Move the cursor.                                 |
| Fire           | Place the previewed tile at the cursor.              |
| Execute        | **Swap** the preview with the stash (one slot).      |
| Cancel         | Exit.                                                |

### Closing & reward
There's no formal "win" state — close the game when you're done.
The session's score scales the inspiration relief to the pet (more
score → more relief), so playing well matters.

## BornJeweled

A trademark-safe Bejeweled-style match-3 swap game on a 6×6 grid,
30-move limit per session.

### Mechanic
Swap two adjacent gems.  Any **3+ in a row or column** clears,
gravity drops the gems above, new gems fall in from the top, and
any matches the cascade creates score with a **chain multiplier**.

### Six gem shapes
All shapes are black on white — accessible by shape alone, no
red-vs-black contrast needed:

| Gem | Shape          |
| --- | -------------- |
|  1  | Filled circle  |
|  2  | Hollow circle  |
|  3  | Filled square  |
|  4  | Hollow square  |
|  5  | Filled diamond |
|  6  | Filled triangle|

### Scoring
**Per run** (within a single cascade tier):

| Run length | Points |
| ---------- | -----: |
| 3          |     30 |
| 4          |     60 |
| 5          |    100 |
| 6          |    150 |

**Per chain step** (a cascade triggered by gravity refill):

| Chain step          | Multiplier |
| ------------------- | ---------- |
| 1 (the swap match)  | 1×         |
| 2 (first cascade)   | 2×         |
| 3 (second cascade)  | 3×         |
| N                   | N×         |

So three 3-matches in a chain pay 30·1 + 30·2 + 30·3 = **180**, not
90.  A lucky 6-row clear in the third tier pays **150·3 = 450** by
itself.

### Controls
| Button         | Effect                                                |
| -------------- | ----------------------------------------------------- |
| Up/Down/Left/Right | Move the cursor.                                  |
| Fire           | First press: select the gem under the cursor.  Second press on an *adjacent* gem: swap. |
| Execute        | Same as Fire (mirror binding).                        |
| Cancel         | Exit early (still scores any inspiration earned).     |

### Closing & reward
Hits 30 moves → "Game Over / Score: N / Press Fire" overlay.  Press
Fire to close.  Score-scaled inspiration is awarded on close
(`add_drained_relief(score / 6)`, capped at 4096).

## Footnote — feature gating

The whole `game` module is behind `#[cfg(feature = "game")]`.  The
`embassy-game` and `embassy` (full) builds include it; the
`embassy-watch` and `embassy-mesh` builds don't.
